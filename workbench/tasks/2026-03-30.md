## claude-火稚鸡 bug fix + test — 2026-03-30 ~15:30-16:00 EDT

### Bug fixed: 采集数据卡在最后一题

**根因**：两个叠加问题：
1. `VLLMServerModel` 的 `openai.OpenAI()` 客户端无 HTTP timeout（默认 600s），vLLM 高负载时最后几题 HTTP 请求永久挂起
2. `process_dataset` process pool 路径无 per-task 超时逃逸，`while remaining_futures` 永不退出；`with ProcessPoolExecutor` 退出时 `shutdown(wait=True)` 进一步死锁

**修改文件**（branch `claude/vibrant-jackson`，commit `3648b66`）：
- `AgentDistill/src/smolagents/models.py`：`VLLMServerModel.__init__` 新增 `timeout=120` 传入 HTTP 客户端
- `AgentDistill/exps_research/unified_framework/processors/base.py`：新增 `per_task_timeout=300` 参数；process pool 改用 `try/finally + shutdown(wait=False, cancel_futures=True)`；每轮 poll 前检查超时 future 并写 error 条目继续

**测试**：Qwen3-32B，MATH 500题，seeds 57-60，gpu008（4xA100）
- seed 57/58/59：500/500 正常完成
- seed 60：500/500 完成，并在日志中确认触发了 bug（`process did not exit after 30s`），fix 正确处理，未死锁
- 全部 4 个 seed 干净结束，GPU 归零
